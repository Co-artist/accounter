## 目标
- 在“首页 / 统计 / 分类 / 我的”四个主页面中，除了点击底部导航外，支持左右滑动切换相邻页面，手势体验一致、无外部库依赖。

## 技术方案
- 采用原生触摸事件实现轻量滑动手势（`touchstart / touchmove / touchend`），不引入第三方库。
- 提供一个可复用的手势指令或组合式函数，用于在四个页面的根容器上挂载。
- 根据滑动方向与距离触发路由跳转到相邻 Tab（保持现有 `vue-router` 结构与 `BottomNavigation` 高亮逻辑）。

## 实现步骤
1. 新增手势工具
   - 创建 `src/utils/swipeNavigation.js`（或指令 `src/utils/swipeNavDirective.js`）：
     - 参数：`currentId`（'home' | 'statistics' | 'category' | 'profile'）、`router`、可选阈值配置。
     - 逻辑：记录 `startX/startY` → 计算 `deltaX/deltaY` → 满足水平滑动阈值（如 |deltaX|≥60 且 |deltaX| > |deltaY|×1.5）时触发。
     - 跳转映射：
       - 左滑（向左）→ 下一个：home→statistics→category→profile
       - 右滑（向右）→ 上一个：profile→category→statistics→home
     - 边界处理：首尾页面滑动不跳转。

2. 集成到页面
   - 在 `HomePage.vue / StatisticsPage.vue / CategoryPage.vue / ProfilePage.vue` 的外层内容容器（如 `.content-wrapper` 或页面根 `div`）上挂载该手势：
     - `setup` 中通过 `ref` 获取容器元素并在 `onMounted` 绑定事件，`onUnmounted` 移除事件。
     - 传入当前页面 ID（例如 Home 为 `home`）。

3. 动效与体验
   - 初版仅做无动画切换，确保稳定；后续可加 CSS 过场（route transition）。
   - 为避免与垂直滚动冲突，使用水平/垂直位移比值判断，微小斜向移动不触发。
   - 设置最小滑动冷却时间（如 300ms）避免连跳。

4. 兼容与细节
   - 保持底部导航现状，点击与滑动任一方式都更新路由，高亮自然跟随。
   - 移动端状态栏/安全区适配不变，不影响现有 `padding-top` 与 `BottomNavigation`。
   - 不改动路由结构；仅在页面层挂载手势。

5. 验证方案
   - 本地运行验证四页间的左右滑动切换与边界页无误跳。
   - Android 真机测试：滑动不影响页面纵向滚动；底部导航点击与滑动互不冲突。
   - 通过现有热更新通道发布前端改动（更新版本号并上传 zip），在手机端自动拉取。
