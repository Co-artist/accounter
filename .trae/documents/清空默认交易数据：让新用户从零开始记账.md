用户的需求是：新用户第一次使用时，应用应该没有任何交易记录（空白状态），而不是像现在这样显示一堆默认的演示数据。

### 问题分析
在 `src/store/index.js` 中，我发现了一段 `defaultTransactions` 代码，这就是导致新用户看到“假数据”的原因。
```javascript
  // 默认交易数据
  const defaultTransactions = [
    {
      id: '1',
      type: 'income',
      amount: 5000,
      // ...
    },
    // ...
  ]
```
代码逻辑是：如果 `localStorage` 里没有数据，就加载这些默认数据。

### 解决方案
我将把 `defaultTransactions` 修改为 **空数组 `[]`**。
这样新用户打开应用时，交易列表就是空的，符合“新用户”的直觉。

同时，用户提到了“生成一段功能演示的动画”，但这是一个较大的新功能。考虑到用户的主要痛点是“不想看到假数据”，我将优先执行“清空默认数据”的操作。关于“演示动画”，我会在清空数据后建议用户是否需要我进一步实现。

**修改文件：**
*   `src/store/index.js`: 将 `defaultTransactions` 设为空数组。
