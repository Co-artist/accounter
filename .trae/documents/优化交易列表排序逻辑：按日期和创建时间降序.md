用户的需求是将“最近交易”按照创建时间排序，目前的现象是同一天内的交易顺序随机（实际上是按照添加顺序，但因为没有明确的排序逻辑，可能不符合预期）。

### 解决方案
目前的排序逻辑仅按照 `date`（日期）排序。
```javascript
return new Date(b.date) - new Date(a.date)
```
当日期相同时（例如都是今天），排序结果是不确定的或仅依赖于数组原有顺序。

由于在 `TransactionForm.vue` 中，新交易的 `id` 是通过 `Date.now().toString()` 生成的，因此 `id` 本身就代表了创建时间。
我将修改 `src/components/TransactionList.vue` 中的排序逻辑，增加**二级排序**：
1.  首要排序：按 `date`（交易日期）降序。
2.  次要排序：按 `id`（创建时间戳）降序。

这样，同一天的交易，最新创建的（ID 更大的）会排在最前面。
