## 目标

* 为主页面切换加入“滑动+淡入淡出”的组合效果，切换更柔和，方向一致（左/右）。

## 技术方案

* 继续使用 Vue `<transition>` 包裹 `router-view`，定义新的过渡名：`slide-fade-left` / `slide-fade-right` / `fade`。

* CSS 同时控制 `transform` 与 `opacity`，并使用轻微位移（进入：±30%；离开：±15%）与 220ms ease-out，提升顺滑度。

* 过渡模式使用 `mode="out-in"` 避免重叠；保留 `will-change` 优化。

* 保持现有方向状态 `navDirection`，仅将名称映射到新的组合效果。

* 可选：尊重系统“减少动态效果”（`prefers-reduced-motion`）自动降级为纯 `fade`；提供用户设置开关后续迭代。

## 实现步骤

1. 更新 `src/App.vue`

   * 将 `<router-view>` 包裹为：`<transition :name="transitionName" mode="out-in">`

   * 在 `setup` 计算 `transitionName`：`left→'slide-fade-left'`，`right→'slide-fade-right'`，默认 `'fade'`。

   * 添加/替换 CSS：

     * `.slide-fade-left-*` / `.slide-fade-right-*`：`transition: transform 220ms ease-out, opacity 220ms ease-out;`

     * `*-enter-from`: `transform: translateX(±30%) scale(0.995); opacity: 0;`

     * `*-enter-to`: `transform: translateX(0) scale(1); opacity: 1;`

     * `*-leave-to`: `transform: translateX(∓15%) scale(0.995); opacity: 0;`

     * `.fade-*` 维持轻量透明过渡。

   * 媒体查询：`@media (prefers-reduced-motion: reduce)` 下将过渡时间设为 `0ms` 或仅 `opacity`。

2. 方向驱动保持不变

   * `src/utils/navDirection.js` 无需改动（已提供 `setNavLeft/setNavRight`）。

   * `src/utils/swipeNavigation.js` 和 `src/components/BottomNavigation.vue` 已在跳转前设置方向，无需改动。

3. 验证

   * 本地构建并在 Android Studio 安装到设备，检查滑动与点击两种路径的动画方向与柔和度。

   * 检查边界页（首页/我的）无越界动画。

   * 检查纵向滚动不受影响。

4. 发布

   * 构建前端并打包热更新 zip。

   * 更新后端版本与说明，推送并验证 Render 部署；手机端自动拉取后生效。
